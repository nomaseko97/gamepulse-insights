# GamePulse

**Understanding the Emotion Behind Every Match.**

GamePulse is an AI-powered gaming sentiment analysis platform. It ingests player chat in real time, classifies each message into six emotional categories (positive, excited, neutral, frustrated, negative, toxic), and surfaces the results through a live moderation dashboard with toxicity heatmaps, player reputation scores, behavior-aware matchmaking, and AI-generated summaries.

## Features

- **Real-time chat feed** — streaming player messages with emotion label and AI confidence score.
- **Six-class sentiment model** — positive, excited, neutral, frustrated, negative, toxic.
- **Per-game categorization** — every message is tagged with its game (Valorant, CS2, Apex, Fortnite, LoL, Dota 2, EA FC 25) and genre (FPS, MOBA, Battle Royale, Sports).
- **Filter bar** — slice all dashboard widgets by game, genre, region, and free-text search.
- **Sentiment trend** — 24-hour stacked area chart of emotion volume.
- **Emotion distribution** — radial breakdown of the active emotional mix.
- **Toxicity heatmap** — region × hour intensity grid.
- **Moderation alerts** — auto-flagged incidents with severity, action taken, and per-game/region context.
- **Player reputation** — leaderboard with reputation score, trend, toxicity rate, badge tier, and main game.
- **Smart matchmaking** — behavior-aware queue panel that ranks lobbies by player quality.
- **AI moderation summary** — natural-language headline + bullet insights, regenerated on demand via *Run AI Sweep*.
- **Functional navigation** — Dashboard, Players, Channels, Reports pages, all live.
- **Functional global search** — search bar in the header drives the same filter context as the FilterBar.

## Getting started

Prerequisites: [Bun](https://bun.sh) (the project uses `bunfig.toml`).

```bash
bun install
bun run dev
```

The dev server starts on Vite's default port and the route tree is auto-generated by the TanStack Router Vite plugin — do not hand-edit `src/routeTree.gen.ts`.

To build:

```bash
bun run build
```

## Filtering & search

A single React context (`src/lib/filter-context.tsx`) holds the active filters: `game`, `gameType`, `region`, `search`. Both the `FilterBar` and the header search bar write into this context, and every data widget (`LiveChatFeed`, `ModerationAlerts`, `ToxicityHeatmap`, `PlayerReputation`, etc.) reads from it via the `matchesFilters` helper. Result: one filter change re-slices the whole dashboard simultaneously.

## AI Sweep

The **Run AI Sweep** button in the header (and the *Regenerate* button on the AI Moderation Summary card) calls `runSweep()` on the filter context. After a short simulated processing delay, `generateAISummary()` produces a fresh natural-language headline + bullets scoped to the currently active filters (game / genre / region). A toast confirms completion.
